## 1. Trade Classification Framework

Every signal TradePro generates should be **classified on three axes** before being shown to the user. This is the missing layer that makes signals actionable.

### 1.1 Horizon classification

| Horizon | Definition | Exit expectation | TradePro track |
|---|---|---|---|
| Intraday | Open and close same session | Same-day mandatory exit | Track B |
| Swing | 2–10 days | Pre-defined target or trail | Track A |
| Medium-term | Weeks to months | Fundamental thesis-driven | Track A |
| Long-term | Months to years | Mandate/allocation driven | Track A (future) |

**Implementation:** Every signal output must include a `horizon` field. Strategies must declare their horizon at registration. A swing strategy must never fire as an intraday signal and vice versa.

### 1.2 Strategy type classification

Observed across today's trades — four distinct strategy archetypes seen at institutional level. TradePro should classify its signals into the same taxonomy:

| Strategy type | What it looks like | Signal inputs | TradePro readiness |
|---|---|---|---|
| **Momentum / trend following** | Buy breakouts, sell breakdowns | Price, SMA, Ichimoku, volume | ✅ Exists today |
| **Mean reversion** | Buy dips, sell rips within range | RSI, Bollinger, deviation from mean | ✅ Exists today |
| **Event-driven** | Trade around earnings, new issues, catalysts | Earnings dates, corporate actions, news sentiment | ❌ Gap — no catalyst awareness |
| **Relative value / pairs** | Long A short B same sector | Correlation, spread, sector beta | ❌ Not built |
| **Index rebalance / forced flow** | Trade ahead of known rebalance events | Index membership changes, float updates | ❌ Not built |
| **Liquidity hunting** | Work large orders over time (like MORLD.OL) | ADV, spread, order size vs float | ❌ Not built — and not appropriate for TradePro's scope |

**Near-term action:** Tag each existing TradePro strategy as Momentum or MeanReversion. Display the tag on signal output. This is cosmetic but builds the classification habit.

### 1.3 Conviction classification

| Level | Criteria | Action |
|---|---|---|
| HIGH | All horizon signals agree + trend filters pass + volume confirms | Full signal, sized normally |
| MEDIUM | Majority agree, one horizon dissents | Signal with reduced sizing suggestion |
| LOW | Conflicting signals, trend filters failing | WATCH only — no entry recommendation |
| INVALID | Signal coherence bug detected (BUG-001/002) | Block display, surface error |

**This directly addresses BUG-001** — a signal with failing trend filters should be capped at MEDIUM or LOW conviction, never displayed as HIGH.

## 3. Intraday Exit Framework

**Core principle:** Every intraday trade entry generated by TradePro must have three mandatory exit parameters computed at signal time. Without all three, the signal is not actionable and must not be displayed.

```
Entry price (P)
  │
  ├── Stop loss:    P × (1 - stop_pct)     ← mandatory, computed at entry
  ├── Take profit:  P × (1 + target_pct)   ← mandatory, computed at entry
  └── Time exit:    Market close - 15min    ← mandatory, hard coded per exchange
```

### 3.1 Default thresholds by strategy type

These are starting defaults for paper trading. They will be tuned from paper trade results.

| Strategy type | Stop loss | Take profit | Ratio | Time exit |
|---|---|---|---|---|
| Momentum breakout | 1.5% | 3.0% | 1:2 | 15min before close |
| Mean reversion (RSI) | 1.0% | 2.0% | 1:2 | 15min before close |
| High volatility name | 2.5% | 5.0% | 1:2 | 15min before close |
| Futures (Phase 4C) | 0.5% | 1.0% | 1:2 | Roll-adjusted close |

**Rationale for 1:2 minimum ratio:** Any strategy with a worse than 1:2 reward/risk needs a >50% win rate to be profitable. Setting 1:2 as the floor ensures even a 40% win rate is net positive.

### 3.2 ATR-adjusted stops (improvement over fixed %)

Fixed percentage stops are blunt. The professional approach (and what institutional desks use) is **ATR-based sizing**:

```
ATR_14 = 14-period Average True Range
Stop loss  = Entry - (ATR_14 × atr_multiplier)   # default: 1.5
Take profit = Entry + (ATR_14 × atr_multiplier × rr_ratio)  # default: × 2.0
```

**Why this is better:** A 1.5% stop on a low-vol name like JNJ is too wide. The same 1.5% on APLD is too tight. ATR naturally scales the stop to the instrument's actual volatility. This is a Phase 4B improvement — compute ATR_14 from existing OHLCV data (already available), derive stops dynamically.

### 3.3 Position sizing from stop distance

Once stop distance is known, position size follows automatically:

```
Max loss per trade  = Account size × risk_per_trade_pct   # e.g. 1% of paper account
Stop distance (£)   = Entry price × stop_pct
Position size       = Max loss per trade ÷ Stop distance
```

Example: £10,000 paper account, 1% risk = £100 max loss. Stop at 1.5% below entry (£1.50 on a £100 stock). Position size = £100 ÷ £1.50 = 66 shares.

**This should be computed and displayed with every intraday signal** — not left to the user to calculate.

### 3.4 Forced close mechanism (the "booking" requirement)

For intraday positions, TradePro must:

1. At signal generation: record entry time, stop, target, and time exit deadline
2. At 15 minutes before close: if position still open → generate mandatory CLOSE signal
3. The CLOSE signal is not optional — it overrides any open P&L state
4. Log the exit reason: `STOP_HIT`, `TARGET_HIT`, or `TIME_EXIT`

**This is what was meant by "always book hedging trades"** — every intraday open position has a guaranteed close. No intraday trade becomes an overnight accident.

### 3.5 Trailing stop option (medium-term swing)

For swing positions (Track A), a trailing stop is more appropriate than a fixed exit:

```
Trail distance = ATR_14 × 2.0  (wider than intraday)
On each new daily close: if price > previous high → raise stop floor
Never lower the stop floor
```

This captures extended moves while protecting against reversal. Implement as an optional flag on swing signals: `exit_style: TRAIL` vs `exit_style: FIXED`.
